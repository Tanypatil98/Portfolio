3.BACK GROUND

An XML document consists of three kinds of tokens: tags, attributes and data values. Consider the following example:

<stuff> <users> <user x="2"> <id>001</id> <name>Chuck</name> </user> <user x="7"> <id>009</id> <name>Brent</name> </user> </users> </stuff>

Here stuff ,users, user,id,name are tags. Each tags occurs in pair (<stuff>) opening tag and (</stuff>) closing tag. The text between the tags are called data values or element contents. The data values are always string but if in other form they are encoded with specific data types. Tags may also have attributes associated with it like here we have “x” attribute associated with (user) tag.XML document works on a structure of a tree, nodes are labelled with tags or attributes and leaves are labelled with data values.

How Compression Happens?

Compression in XML file occurs when the spaces are removed because spaces occupy more memory in file and hence increases the size of file, so by removing spaces we can compress the XML file. But in our compression we are not only removing the space content from file but also removing the repeated tags that occurs in XML document which will further reduce more memory as the repeated characters are been removed and thus making the file more optimal one.

{"user": [{"@x": "2", "id": "001", "name": "Chuck"}, {"@x": "7", "id": "009", "name": "Brent"}

Here user tag was repeating two times in original XML file which was reduces to one in json converted format which maps both dictionaries hence resulting that the tag is related

with both dictionaries inside that key.

4.WORKING FLOW OF COMPRESSION

Our compression techniques involves following stages from where XML files passes by and thus at each step making that file more optimized and finally giving the compressed and flexible file. Now as we are using json format to convert our original XML file to compressed version. So we will be using python as our parser language. And Json and dictionary libraries are out intermediate converters of data.

STEP 1: import the required libraries needed that will change the datatype format of original XML file. Here we will be using two of the following libraries which will give us the converted form of data.

import xmltodict import json

(i)xmltodict library and its functions: xmltodict is a Python module that makes working with XML feel like you are working with JSON. If xml_attribs is True, element attributes are put in the dictionary among regular child elements, .using @ as a prefix to avoid collisions. If set to False, they are just ignored. This is done by using xmltodict.parse(‘’XML document”) function.

Example:

OrderedDict([('stuff', OrderedDict([('users', OrderedDict([('user', [OrderedDict([('@x', '2'), ('id', '001'), ('name', 'Chuck')]), OrderedDict([('@x', '7'), ('id', '009'), ('name', 'Brent')])])]))]))])

Now this xmltodict when is dumped with json it gives us the required format of our compression.

(ii)json library and its functions : The json library can parse JSON from strings or files. The library parses JSON into a Python dictionary or list. It can also convert Python dictionaries or lists into JSON strings. We are using json.dumps() functions to convert the python dictionary into json strings.
